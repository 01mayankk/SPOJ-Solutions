# SPOJ Solutions

A curated collection of my accepted solutions to **Sphere Online Judge (SPOJ)** problems, implemented in **C++**.

This repository serves as my personal archive of problem-solving solutions and is designed to help me strengthen my understanding of **Data Structures**, **Algorithms**, and **Competitive Programming**. Every solution is written with readability in mind and is accompanied by an explanation of the approach used.

## 📂 Repository Structure

Each problem is organized into its own folder using the official SPOJ problem code.

```text
Problem_Code/
│── Question.md      # Problem statement
│── README.md        # Approach, intuition, dry run & complexity
└── Solution.cpp     # Well-commented C++ solution
```

Example:

```text
EKO/
AGGRCOW/
ONP/
```

## 📖 What You'll Find

For every problem, this repository includes:

* 📄 **Question.md** – Problem statement, constraints, and sample test cases.
* 📘 **README.md** – Intuition, approach, dry run, correctness, and complexity analysis.
* 💻 **Solution.cpp** – Clean, well-commented C++ implementation.

The goal is to understand the reasoning behind each solution rather than simply storing accepted code.
